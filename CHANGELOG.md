# Changelog

DeepCoffee 的重要变更记录。日期为变更落地日。

## 0.33.0 · 2026-06-28

### 记冲煮记录时自动认出系统里已有的器具和豆子

- **滤杯 / 磨豆机 / 豆子会自动对应到已有的，不用再手动重选**：以前把冲煮方案记成记录时，「V60」认不出是你存的「Hario V60 01」、「1Zpresso ZP6 Special」认不出是「ZP6S」、豆名写简称也对不上整名，于是都退回手动选。现在系统会按规范名 + 别名（含简写、缩写、英文）把它们认出来并预选好；豆子还会直接关联到你豆仓里的那支。
- **新增公共器具目录**：内置了一批常见滤杯 / 磨豆机 / 滤纸 / 用水的规范名和别名（可持续扩充）；记录和「我的器具」的下拉里都能直接选目录里的器具。
- **录入器具即归一**：以后把器具存进「我的器具」时，会自动归到目录里的规范名，避免再产生「同一台器具好几种写法」的脏数据。
- **没见过的器具自动进目录**：你用到的器具如果目录里还没有，系统会在后台用 AI 推断它的规范名和别名并直接加进公共目录（标记为「AI 发现」，管理员可在后台审计 / 清理），下次就能直接认出。
- **始终保留手动**：认不出或认错时，下拉的「自定义输入」和手选一直都在，由你兜底。

## 0.32.1 · 2026-06-28

### 聊天里求冲煮方案改为「直接出方案」

- **在聊天里说「给我出个推荐冲煮方案」现在会直接给方案**:过去这类请求会被回一句笼统的「我可以帮你聊咖啡豆、冲煮方案、器具这些」,看起来像答非所问。现在会依据当前这支豆子、你设置的器具和口味偏好,直接给出一版完整方案(粉量 / 水温 / 粉水比 / 分段注水步骤 / 预期风味 / 后续微调方向),纯文字呈现。
- **方案只生成、不自动记录**:给出的方案不会自动计入冲煮记录;只有你明确说「记录这个方案 / 计入冲煮记录」,才会据此生成一张冲煮草稿,确认后入库。
- **顺带修掉一处「答非所问」**:在聊天里用文字说要新建/修改豆卡时,以前也可能落到那句笼统兜底;现在会明确引导你到「豆仓」新建或编辑。

## 0.32.0 · 2026-06-28

### 记录豆卡改为「先录入、再追问是否记冲煮」

- **记录豆卡时不再顺带弹冲煮草稿**:过去说「帮我记录这个豆卡」会被误判成「想记一杯冲煮」,在录入豆卡的同时多弹一张冲煮记录草稿。现在录豆卡就只录豆卡。
- **新增「追问」步骤**:豆卡录入后,先告诉你「已录入豆仓」,再问一句「要顺手记录一次这支豆子的冲煮吗?」,给【记录一次冲煮】【暂不】两个按钮。点「记录一次冲煮」才生成冲煮草稿,且草稿会自动关联刚录入的豆子;若豆袋上有官方推荐参数,会用它预填(没有就留空)。
- **生成建议参数的交互修整**:豆卡详情页「建议冲煮参数」的生成对话改为纯文字展示(不再是聊天气泡);移除了「重新生成」按钮(该功能暂不提供)。

## 0.31.2 · 2026-06-27

### 豆仓广场卡片大小与「我的豆仓」一致

- **广场卡尺寸对齐我的豆仓,翻面信息不再被裁**:上一版广场卡虽改成了翻转卡,但网格是 3 列、卡片比我的豆仓小,翻到背面时「风味强度 / 用户评价」和「建议冲煮参数」会被截断显示不全。现在广场和我的豆仓一样用 2 列布局,卡片大小一致,背面信息完整展示。

## 0.31.1 · 2026-06-27

### 豆仓广场卡片与「我的豆仓」统一

- **豆仓广场的豆卡改成和「我的豆仓」一样的翻转卡**:之前广场卡是单面长卡、处理法还带药丸样式,和我的豆仓不一致。现在统一成 1:1 翻转卡——正面是名称 / 风味 emoji / 产地·品种·处理法(纯文字,拼配多列) / 评分,背面是风味强度(或用户评价)+ 建议冲煮参数。选择勾选放在卡片角落、查看详情在背面,导入和查看功能不变。

## 0.31.0 · 2026-06-26

### 豆卡展示优化 + 知识文章来源修复

- **豆卡整卡按处理法配色**:豆卡正反面、豆仓广场卡的颜色按处理法区分并和站点暖色明显拉开——水洗=清爽蓝、日晒=烈日橙、蜜处理=蜜金、厌氧/二氧化碳浸渍=发酵紫,处理法未知或拼配(多豆源)则用冷调近白。处理法名中英文都能识别(过去英文如「Washed」会落回默认色)。
- **风味标签配 emoji**:每个风味词前面带一个贴切的小图标(荔枝🍒、柑橘🍊、花香🌸、巧克力🍫、坚果🌰…)。内置一份较全的风味词库即时上图标,识图/解析豆卡时还会由 AI 给词库里没有的新风味即时配图标。
- **豆卡正面三属性统一**:产地、品种、处理法统一为同款纯文字展示(过去处理法是药丸样式);拼配多豆源时按豆源分多列展示。
- **豆卡详情「豆源」更清爽**:去掉每条豆源外层的方框和「豆源 1/2…」序号,直接列出豆源信息,多豆源多行展示。
- **豆卡背面去掉「烘焙商维度」标注**;当这支豆没有风味强度数据时,改为展示你给的评分(总评/风味/酸度…)作为「用户评价」。
- **知识文章「来源」不再留空标题**:当一篇文章的来源只引用了内部素材(不对外展示)时,过去会在页面留下一个下面空空的「来源」标题,现在这种情况会把标题一并隐藏。

## 0.30.1 · 2026-06-26

### 对话：刻度换算问句不再误弹冲煮草稿

- **问研磨刻度换算时不再附冲煮草稿卡**:过去只要消息里出现「刻度」或「磨豆机名 + 数字」,系统就会误判成「在报冲煮参数」、主动塞一张冲煮草稿卡。比如问「C40 刻度 22 对应 ZP6s 是多少」时,除了正常的换算回答还会多出一张草稿卡。现在改为按「冲煮记录的必要元素」判定:要么你明确说要记录(记录这杯 / 想记录…),要么你确实报了冲煮详情(粉量 / 水量 / 水温 / 时间 / 粉水比 / 分段 凑够两项以上),才会给草稿卡;只提磨豆机和刻度不再触发。

## 0.30.0 · 2026-06-26

### 豆卡：支持拼配 / 多豆源

- **豆卡录入改为「豆源」子表单，单豆与拼配统一**:过去豆卡按「一张卡一支豆」设计,产地/庄园/生豆商/处理法/品种/海拔/采收期都在顶层,遇到拼配(多豆源)产品不知往哪填。现在这些豆子信息统一放进「豆源」——单豆填 1 条、拼配加多条,顶层只留产品级信息(烘焙商/产品名/烘焙日期/净含量)。每条豆源可独立填产地、庄园、生豆商、处理法、品种、海拔、采收期、占比。
- **拼配也能正确聚合 / 派生 / 收录**:豆仓广场「同豆聚合」、冲煮记录的豆名产地派生、公共实体收录,全部改为基于「豆源」——拼配卡的每条豆源都会各自参与同豆判定与实体收录;拼配卡的冲煮记录产地显示为「拼配 · 多产地」。单豆行为不变。

## 0.29.0 · 2026-06-25

### 冲煮记录 / 豆卡

- **冲煮记录的豆名等统一跟豆卡走，改名不再有脏数据**:过去每条冲煮记录会把豆名、产地、烘焙商、处理法、品种各存一份快照,豆卡一改名这份就过期、导致记录在豆卡里关联不上。现在这些信息一律实时取自关联豆卡——改一次豆卡,名下所有记录立即同步,彻底消除这类不一致(按豆名搜索、同豆参考也都跟着对齐)。
- **每条冲煮记录都必须关联一张豆卡**:对话里记录一杯冲煮时,若还没有对应豆卡,保存时会自动为它建一张(用已知的豆名/产地等,缺的留空),不再出现"记了一杯却没挂到任何豆子"的游离记录。

## 0.28.3 · 2026-06-25

### 豆卡 / 冲煮记录

- **豆卡按钮调整**:正面右上角改为「豆卡详情」,反面右上角改为「冲煮记录」;去掉了反面的「参数详情」按钮(查看 / 生成建议参数在豆卡详情页里本来就有,不受影响)。
- **冲煮记录关联改用豆卡 ID,改名不再断链**:从豆卡进入「冲煮记录」时,过去是拿豆卡当前名字去逐条精确比对记录里的豆名——豆卡一旦改名,旧记录因豆名对不上就关联不到、列表里看不见了。现在改为按豆卡的内部 ID 关联(豆卡改名也不影响),只有早期未关联豆卡的老记录才回退按豆名匹配。

## 0.28.2 · 2026-06-25

### 移动端稳定性

- **修复移动端「久久载入 / 卡死」与英文报错**:移动网络偶发抖动时,过去会出现侧栏账户与用量长时间停在「载入中」、页面卡死,或弹出英文 "Fetch is aborted"。现在请求遇到网络问题会快速失败并自动重试一次,超时统一给中文提示;登录态检查加了超时兜底,不再永久卡在「载入中」;账户信息与用量改为各自独立加载(一个慢不连累另一个),并在加载失败处提供「重试」按钮。

## 0.28.1 · 2026-06-24

### 数据治理(公共实体库)

- **种子导入主名统一取中文**:知识库实体标题多写成「中文 / English」双语形式,导入器以前直接拿整条双语串当实体主名;现在自动取其中的中文为主名、英文等写法转为别名(纯英文名维持原样)。从源头杜绝双语主名,并消除「清掉同步台账后重新导入会按双语名匹配不上、误造重复实体」的隐患(此前 Caturra / Maracaturra 等就踩过)。仅影响导入工具,不改线上运行行为。

## 0.28.0 · 2026-06-24

### 豆仓广场

- **同款豆只显示一张(同豆聚合)**：多个用户各自录入同一支豆时,广场不再重复罗列,而是聚合成一张卡——显示「多少人在喝」、综合评分,以及各家的匿名评论汇总。同款判定按 烘焙商 + 名称 + 产地 + 处理法(烘焙商/产地/处理法已按公共实体归一,不同写法自动认同);不同豆仍各自展示。
- **豆卡关联烘焙商产品实体**：建卡 / 编辑时,豆卡会在「该烘焙商名下」匹配并关联到已收录的烘焙商产品实体(只连已审核通过的,不自动新建),为后续更精准的同豆识别打底。

### Dashboard 新手引导

- **新增可关闭的新手任务条**：Dashboard 顶部按顺序引导新用户完成核心流程——① 建第一张豆卡 ② 添加常用器具 ③ 记录一次冲煮 ④ 获取冲煮建议(完成前 3 步后解锁);任务条按豆卡 / 器具 / 冲煮记录数量自动判断完成状态,完成显示绿色勾选与进度,点「不再显示」可关闭(#28)。

## 0.27.0 · 2026-06-23

### 数据治理(公共实体库)

- **产品实体只留具体单品**:清掉混进来的「订阅 / 系列 / 项目」类条目(如 April 订阅、Friedhats Filter Series、Royal Crown Jewels 选品等),烘焙商产品与生豆商产品实体只保留具体单品;种子导入器加守卫,以后这类产品线不再被当作产品实体收录。
- **修复两处错误合并**:Caturra(卡杜拉)曾被误并进 Maracaturra(马拉卡杜拉)、印度尼西亚曾被误并进印度——已还原为各自独立实体,搜索 / 识别恢复正确。
- **补全缺失实体**:新增 Let's Grind、Coffee Buff(加福咖啡)、Tabi、SL9、Caturron(卡杜隆)等实体;「原生种 / Heirloom」并入埃塞俄比亚地方种、「Konga / 孔加」并入 Konga Cooperative;并对历史豆卡重跑实体关联,让其正确连上公共实体。

## 0.26.1 · 2026-06-23

### 豆仓广场

- **广场只展示原创豆卡**：从广场「加入我的豆仓」产生的副本不再出现在豆仓广场,避免同一支豆随导入次数重复堆叠;副本只存在于各自的「我的豆仓」。

## 0.26.0 · 2026-06-23

### 会员与权益

- **套餐分 Basic / Pro / Max**：每月 AI 问答次数分别 99 / 500 / 1000;用户端只显示用量百分比,后台显示真实已用次数与上限,可直接切换套餐。

### 知识库权限

- **知识库自由浏览仅 Max 开放**;Basic / Pro 只能打开 AI 曾经引用过的文章——问答命中某篇文章后,自动获得该篇的查看权。

### 同豆冲煮参考

- **同豆匿名冲煮记录**：Pro / Max 在自己已有该豆豆卡的前提下,可查看其他用户冲煮同一支豆的匿名记录(只含咖啡相关字段,不含身份 / 原始输入 / 备注)。

### 豆仓广场

- **新增「豆仓广场」**(Pro / Max)：浏览匿名豆卡及作者的公开评论,可单选 / 多选「加入我的豆仓」——为自己创建一份私人副本(不复制原作者评论 / 备注,重复加入不会重复创建);Basic 访问见升级提示。

### 豆卡

- **新增公开「评论」字段**：评论会匿名展示在豆仓广场;原有「备注」保持仅本人可见。

## 0.25.0 · 2026-06-21

### 豆仓

- **豆卡信息更贴近真实豆袋**：豆卡新增海拔、采收期、烘焙日期、净含量和豆源组成，拼配 / 多产地产品可以按多条豆源记录。
- **主名称创建后固定**：豆卡名称作为核心识别信息，编辑页不再允许修改；新建时可用烘焙商产品 / 批次名作为主名称。
- **保存规则更清晰**：烘焙商、产地和处理法成为必填项；用户填写的新烘焙商、产地或处理法仍可先保存到个人豆卡，并进入后台审核候选。
- **风味信息不再显示占位维度**：没有豆袋 / 烘焙商提供的风味维度时，不再展示默认酸质、甜感等圆点；评分区域去掉内部同步说明。
- **编辑页改为单列表单**：编辑态只展示可编辑内容，建议冲煮参数放在备注前；查看冲煮记录、记录一次冲煮和推荐参数聊天只在查看态出现。

### 数据迁移

- **新增豆袋字段迁移脚本**：新增 `user_bean_cards.altitude_text`、`harvest_date_text`、`roast_date_text`、`net_weight_text` 和 `bean_components`。

## 0.24.0 · 2026-06-20

### 冲煮记录

- **豆子评分和冲煮评分拆开**：豆子评分成为豆卡属性，所有同豆冲煮记录读取同一份评分；冲煮记录新增「本次冲煮评分」，只影响当前记录。
- **新增表单创建记录**：「新建记录」现在可选择 AI 对话新增或表单直接新增；表单新增必须关联豆卡，不消耗 AI 额度。
- **编辑页改为完整单列表单**：编辑时只展示可编辑内容，冲煮参数、感官记录、冲煮阶段、豆子评分和本次冲煮评分都可编辑；原始输入、AI 复盘、同豆对比和问 AI 仅在展示态出现。
- **冲煮阶段可编辑**：可按序号、用时、水量和手法维护阶段列表。
- **粉水比自动生成**：粉水比由粉量和水量计算，不再直接编辑；缺少任一项时留空。
- **列表去掉器具列**：列表改为豆子、参数、水温、冲煮评分，器具信息保留在豆子下方摘要，避免重复和列错位。

### 豆卡

- **豆卡支持 7 项评分**：总评、香气、风味、余韵、酸质、触感、平衡度都可在豆卡或关联冲煮记录中维护，并同步到同豆记录。

### 数据迁移

- **新增评分迁移脚本**：新增 `user_bean_cards.rating` 和 `brew_records.brew_score`；旧冲煮记录 `evaluation` 保留兼容，并在无冲突时迁入豆卡评分，若同豆存在多份旧评分会停止并输出冲突清单。

## 0.23.1 · 2026-06-20

### Coffea 图片问答

- **修复移动端带图短问答答非所问**：用户上传图片后问「这个水冲咖啡合适吗」这类依赖图片的问题时，Coffea 会稳定进入看图回答路径，不再回通用追问。
- **图片知识问答兜底更稳**：即使知识库没有命中相关条目，只要本轮问题明显依赖图片，后端仍会调用视觉模型结合图片和咖啡经验回答，并提示判断依据有限。
- **移动端上传保护**：图片还在压缩 / 读取时，聊天输入栏会显示「图片处理中…」并暂时禁用发送，避免文字先发出而图片没进请求。

## 0.23.0 · 2026-06-19

### Coffea 草稿卡状态

- **保存 / 忽略状态可恢复**：聊天里的豆卡、冲煮记录、器具草稿在确认保存或忽略后，会把状态写回服务端聊天历史；切换页面或重新进入聊天页后仍显示「已保存」或「已忽略」，不再退回原始草稿提示。

- 厂商 key 配额/欠费导致的降级显式可见：用户回复末尾追加「AI 服务暂时不可用，本次为基础回复。」，服务端记 ERROR（Sentry 告警）。
- `/me/quota`、`/billing/usage`、侧边栏与设置页改读自管额度。

### 管理后台：用户管理重构 + 操作审计

- 用户列表改纯展示，编辑收纳进「操作」列：「详情」弹窗（套餐 / 角色 / 禁用恢复 / 额度调整）、「修改历史」弹窗。
- 新表 `admin_audit_events`（迁移 `008`）：管理员对用户的每次字段变更（套餐 / 角色 / 状态 / 月上限 / 已用数）各留一条审计，含变更前后值、原因与操作人；新端点 `GET /admin/users/{id}/audit`。

### 对话豆卡识图闭环

- 识图结果不再裸显内部动作名与原始 JSON：高识别度（综合识别度 ≥ 阈值，默认 0.8，可配）直接自动建档并以中文回复确认；低识别度在对话内出可编辑草稿卡，确认后保存；同名豆防重；发错图 / 识别失败给出手动录入引导。
- 识别度取 vision 自报与字段完整度评估的较小值，防模型自报过高。

### 冲煮建议体验

- 器具支持「默认套」（`is_default`，迁移 `006`）：首套自动设默认，器具页可切换；有默认套时生成建议首轮直接出参数，不再追问。
