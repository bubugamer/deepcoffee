# Changelog

DeepCoffee 的重要变更记录。日期为变更落地日。

## 2026-06-11

### 重大：剥离 new-api，额度自管 + 模型直连

- 移除 new-api 网关全链路（客户端、影子计费账户、修复接口、容器编排）。
- AI 额度改为 DeepCoffee 内部自管：`ai_usage_events`（真实调用）+ `ai_usage_adjustments`（管理员调整）+ `user_ai_quota_settings`（每用户月上限），按自然月窗口计数。迁移 `007`。
- 模型调用改为后端直连 OpenAI 兼容接口（`DEEPCOFFEE_MODEL_*`），vision 独立通道（`DEEPCOFFEE_VISION_MODEL_*`）；厂商 key 只存在服务端，不再为用户创建任何厂商侧账户。
- 计费口径：一条用户请求只计一次额度（只统计入口动作；同一条消息的能力执行明细不重复计数）。
- 厂商 key 配额/欠费导致的降级显式可见：用户回复末尾追加「AI 服务暂时不可用，本次为基础回复。」，服务端记 ERROR（Sentry 告警）。
- `/me/quota`、`/billing/usage`、侧边栏与设置页改读自管额度。

### 管理后台：用户管理重构 + 操作审计

- 用户列表改纯展示，编辑收纳进「操作」列：「详情」弹窗（套餐 / 角色 / 禁用恢复 / 额度调整）、「修改历史」弹窗。
- 新表 `admin_audit_events`（迁移 `008`）：管理员对用户的每次字段变更（套餐 / 角色 / 状态 / 月上限 / 已用数）各留一条审计，含变更前后值、原因与操作人；新端点 `GET /admin/users/{id}/audit`。

### 对话豆卡识图闭环

- 识图结果不再裸显内部动作名与原始 JSON：高识别度（综合识别度 ≥ 阈值，默认 0.8，可配）直接自动建档并以中文回复确认；低识别度在对话内出可编辑草稿卡，确认后保存；同名豆防重；发错图 / 识别失败给出手动录入引导。
- 识别度取 vision 自报与字段完整度评估的较小值，防模型自报过高。

### 冲煮建议体验

- 器具支持「默认套」（`is_default`，迁移 `006`）：首套自动设默认，器具页可切换；有默认套时生成建议首轮直接出参数，不再追问。
- 研磨建议给出具体刻度区间：内置常见磨豆机刻度参考表（C40、ZP6/ZP6S、1zpresso、泰摩、Kingrinder、EK43 等），未覆盖的型号由模型给「区间 + 调整指引」，不认识才退回相对描述；同步沉淀知识库文章《磨豆机刻度对照表》。
- 豆卡建议参数卡片「器具」label 更正为「滤杯」。

### 对话体验

- 发送中可停止（前端中断等待）；用户头像取账户信息；历史会话本地持久化（刷新不丢）。
- 新增「我的器具」页面（对话录入与手动维护共用一张表）。

## 2026-06-10

### 初始版本

- 全栈首发：FastAPI 后端 + Next.js 前端 + Supabase（Postgres / Auth）+ Markdown 知识库。
- Coffea 统一对话入口：意图调度（模型路径 + 本地兜底）、知识库问答、图片理解、冲煮教练、联网核实（Brave Search）。
- 豆仓（解析 → 确认建档、动态风味维度）、冲煮记录（AI 解析 / 复盘 / 同豆对比）、建议参数多轮闭环。
- 邀请制注册 + 引导邀请码初始化首个管理员；管理后台（概览 / 邀请码 / 用户 / 审核 / 知识库）。
- 公共实体库与候选事实审核链路；知识库实体种子导入。
