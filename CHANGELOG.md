# Changelog

DeepCoffee 的重要变更记录。日期为变更落地日。

## 0.17.0 · 2026-06-17

### 实体库整理

- **产地 / 庄园显示名规范为单一主名**：24 个产地（如「哥斯达黎加（Costa Rica）」→「哥斯达黎加」）与 2 个庄园（如「Finca Santo Niño（圣尼佐庄园）」→「Finca Santo Niño」）的中英混写主名收成一个干净主名，另一种写法转入别名，照样能搜到；知识库对应文章标题同步变干净。

## 0.16.0 · 2026-06-17

### 对话体验

- **聊天图片可点击放大与下载**：聊天里上传的图片原先只有小缩略图、点了没反应；现在点击可弹出全屏大图看清内容，并在大图里一键下载原图（点遮罩 / X / Esc 关闭）。

## 0.15.0 · 2026-06-17

### 修复：AI 恢复真正作答

- **修复对话 AI 不作答、只罗列参考资料的问题**：此前因思考类模型的兼容性问题（思考模式过慢、占满输出额度、还限制 temperature），每次模型调用都失败、全程回退到本地兜底——只列出知识库文章、读不了图片、不真正回应用户。现已让这类模型关闭思考模式并适配其参数，对话 / 知识问答 / 联网核实 / 识图恢复由模型真正作答。
- **带图问「点评这个冲煮方案」能正确点评**：这类请求改由会看图的冲煮教练处理（原先被误判为粉床照片分析、产出空回复），现在能结合图片逐条点评方案、给出改进建议。

## 0.14.0 · 2026-06-16

### 体验修复

- **问候语按时区显示**：首页问候语改为按账号时区的当前时间显示（夜深了 / 早安 / 中午好 / 下午好 / 晚上好），不再固定「早安」。
- **点 Logo 回到首页面板**：登录后点左上角 DeepCoffee 标志回到「首页面板」（早安页），而不是未登录落地页，避免像是掉了登录。
- **新增退出登录**：侧边栏底部用户区新增「退出登录」按钮，登出后清理本地凭证与 Supabase 会话并回到登录页，登录状态闭环。

## 0.13.0 · 2026-06-16

### Coffea 对话质量

- **咖啡问题优先查内部知识库**：知识 / 事实类问题默认走知识库，只有用户明确要「网上说法 / 最新 / 口碑 / 核实」或知识库确实没有时才联网核实，不再动辄去爬第三方社区。
- **见解归入回答、卡片只列来源**：联网核实 / 知识库的综合判断统一放进主回复，「参考来源」卡片只保留链接，不再在正文里重复罗列来源；一轮里有多条回答（如知识库答一部分 + 联网核实补一部分）会合并成一段完整回复。
- **修复无关兜底语顶替真实回答**：此前偶发把通用的「冲煮教练」兜底引导当成主回复、把真正的答案挤进卡片，现已纠正。
- **磨豆机刻度对照表新增「MAVO 幻刺 Pro」**：补充手冲 / 意式刻度与归零方式，并给出与 C40 的换算参考。

### 实体名多语言（地基）

- 公共实体的别名按语言（中 / 英 / 日）打标，接口可按用户语言返回显示名、取不到回退主名；去重与匹配能力不变。完整规划见 `docs/国际化计划.md`。

## 0.12.0 · 2026-06-16

### 实体消歧系统（公共实体库去重）

把「同一个烘焙商 / 产地 / 处理法的不同写法」收敛到同一个公共实体，止住重复条目，并让私有豆仓也能按实体聚合搜索。

- **录入即归一（止住新重复）**：新建公共实体时自动登记别名（中英拆分、大小写 / 空格 / 全半角形态归一）；此后任何写法录入都先比对「主名 + 全部别名」，命中即复用、不再重复建档。形态差异自动认同，缩写 / 中英等拿不准的交人工。
- **审核「疑似并入」**：候选事实审核时提示「疑似已有实体」（含缩写↔全称，如 SEY ↔ SEY Coffee），管理员可一键并入——候选写法登记为该实体别名，不新建实体。
- **私有豆卡关联实体 + 搜索聚合**：建档 / 编辑豆卡时把烘焙商等名称解析到公共实体并回填关联；豆仓搜索某烘焙商时，其不同写法（如 coffee buff / Coffeebuff）的豆卡会被一起搜出。
- **管理员清理工具**：实体库新增「清理建议」——疑似重复实体一键合并（引用与别名整体迁移到保留实体，旧实体标记 merged）、中英混合主名一键规范成单一主名（旧名转别名仍可被搜到）。

## 0.11.0 · 2026-06-15

### 对话记忆系统

- **记住最近对话**：每轮把最近若干轮对话一起喂给模型，修复「隔一会儿回来像第一次对话」——此前模型每轮只看到当前消息和活跃实体，看不到聊过什么。
- **长期对话摘要**：超出窗口的旧对话不再直接丢弃，后台增量压成「按主题归类、带时间线索」的长期摘要，跨多轮记起。
- **用户画像沉淀**：后台从对话与冲煮记录中抽取稳定的口味偏好与习惯，存入可查可改的「用户记忆」，跨会话个性化；新增 `GET/PATCH/DELETE /coffea/memories`。
- **会话不再断线**：清浏览器缓存等导致 `session_id` 丢失时，后端回退到该用户的永久对话，不再断成空白新会话。
- 摘要与画像抽取都在响应返回后异步进行，不阻塞对话、失败也不影响正常回复。

### 可观测性

- **Langfuse trace 记自然语言 IO + 响应时间**：trace 的 input/output 改记用户实际读到的文本（而非结构化路由计划），并记录每轮处理耗时（#2，感谢 @jackymlr）。

## 0.10.0 · 2026-06-14

### 对话稳定性与体验

- **防卡死**：前端请求加自动超时（普通接口 45s、聊天 90s），网络卡住/请求丢失时给出「超时，请重试」而不再永久转圈。
- **上传前压缩图片**：手机原图（数 MB）先等比缩到长边 ≤1600px、转 JPEG（约 200–400KB），弱网上传更稳更快，识图也更快——解决移动端发图常卡住、桌面端却正常的问题。
- **上传的图片跨设备可见**：聊天图片改存 Supabase Storage 公开桶，会话历史记图片 URL；手机/电脑登录同一账号都能看到自己发过的图（图片本身不进数据库）。
- **识图建卡后顺带给冲煮建议**：上传豆卡并要「方案/建议」时，有默认器具 → 按默认器具直接给一版热冲方案；无默认器具 → 引导去「我的器具」设置。

### 说明

- 品牌歧义消除经核实为按设计：私有豆卡不自动合并，消歧发生在公共实体库的管理员审核链路（候选 → 审核 → 别名/归一）。

## 2026-06-11

### 重大：剥离 new-api，额度自管 + 模型直连

- 移除 new-api 网关全链路（客户端、影子计费账户、修复接口、容器编排）。
- AI 额度改为 DeepCoffee 内部自管：`ai_usage_events`（真实调用）+ `ai_usage_adjustments`（管理员调整）+ `user_ai_quota_settings`（每用户月上限），按自然月窗口计数。迁移 `007`。
- 模型调用改为后端直连 OpenAI 兼容接口（`DEEPCOFFEE_MODEL_*`），vision 独立通道（`DEEPCOFFEE_VISION_MODEL_*`）；厂商 key 只存在服务端，不再为用户创建任何厂商侧账户。
- 计费口径：一条用户请求只计一次额度（只统计入口动作；同一条消息的能力执行明细不重复计数）。
- 厂商 key 配额/欠费导致的降级显式可见：用户回复末尾追加「AI 服务暂时不可用，本次为基础回复。」，服务端记 ERROR（Sentry 告警）。
- `/me/quota`、`/billing/usage`、侧边栏与设置页改读自管额度。

### 管理后台：用户管理重构 + 操作审计

- 用户列表改纯展示，编辑收纳进「操作」列：「详情」弹窗（套餐 / 角色 / 禁用恢复 / 额度调整）、「修改历史」弹窗。
- 新表 `admin_audit_events`（迁移 `008`）：管理员对用户的每次字段变更（套餐 / 角色 / 状态 / 月上限 / 已用数）各留一条审计，含变更前后值、原因与操作人；新端点 `GET /admin/users/{id}/audit`。

### 对话豆卡识图闭环

- 识图结果不再裸显内部动作名与原始 JSON：高识别度（综合识别度 ≥ 阈值，默认 0.8，可配）直接自动建档并以中文回复确认；低识别度在对话内出可编辑草稿卡，确认后保存；同名豆防重；发错图 / 识别失败给出手动录入引导。
- 识别度取 vision 自报与字段完整度评估的较小值，防模型自报过高。

### 冲煮建议体验

- 器具支持「默认套」（`is_default`，迁移 `006`）：首套自动设默认，器具页可切换；有默认套时生成建议首轮直接出参数，不再追问。
- 研磨建议给出具体刻度区间：内置常见磨豆机刻度参考表（C40、ZP6/ZP6S、1zpresso、泰摩、Kingrinder、EK43 等），未覆盖的型号由模型给「区间 + 调整指引」，不认识才退回相对描述；同步沉淀知识库文章《磨豆机刻度对照表》。
- 豆卡建议参数卡片「器具」label 更正为「滤杯」。

### 对话体验

- 发送中可停止（前端中断等待）；用户头像取账户信息；历史会话本地持久化（刷新不丢）。
- 新增「我的器具」页面（对话录入与手动维护共用一张表）。
